Utility Classes : Position

Utility : Position

I. Description

Collection of used to define an element's position.

II. Initialization

Download and refference the asterisk-utility-position.css CSS file.

<link href="--myPath--/asterisk-utility-position.css" type="text/css" rel="stylesheet">

Alternatively, a {*}-spec.css file can be generated within the [Code] Tab

<link href="--myPath--/asterisk-utility-position-spec.css" type="text/css" rel="stylesheet">

III. Usage

Classes are applied to the HTML Element and take effect on the HTML Element itself.

Format :
<div class="pos-{value}"> </div> Format :
<div class="position-{value}"> </div>
Example :
<div class="pos-relative"> </div>
// sets position: relative
Example :
<div class="position-relative"> </div>
// sets position: relative

IV. Class-name Syntax

Each className follows the format : pos-{ value } or position-{ value }.

Longhand Suffix Shorthand Suffix Effect
position-static pos-static position: static;
position-relative pos-relative position: relative;
position-absolute pos-absolute position: absolute;
position-fixed pos-fixed position: fixed;
position-sticky pos-sticky position: sticky;

V. Responsive Utilities : Viewport-Prefixes

To create different scenarios for specific viewports, any class can be prefixed with any *viewport*- or *viewport*-o- syntax.

Format :
<div class="{viewportPrefix}-{className}"> </div>
Example :
<div class="pos-relative"> </div>
// sets position: relative on all viewports
Example :
<div class="sm-pos-relative"> </div>
// sets position: relative on small(sm) viewports or larger
Example :
<div class="sm-o-pos-relative"> </div>
// sets position: relative on small(sm) viewports only
