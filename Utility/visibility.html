Utility Classes : Visibility

Utility : Visibility

Collection of , used to show / hide an element on different viewports.

Download and refference the asterisk-utility-visibility.css CSS file.

<link href="--myPath--/asterisk-utility-visibility.css" type="text/css" rel="stylesheet">

Classes are applied to the HTML Element and take effect on the HTML Element itself.

Format :
<div class="{viewport}-visible"> </div> Format :
<div class="{viewport}-hidden"> </div>
Example :
<div class="xs-hidden"> </div>
// the example hides the element on extra-small viewports only
Example :
<div class="xs-visible"> </div>
// the example hides the element on all viewports except extra-small
{viewport}-hidden Will hide the Element on the specified viewport only
{viewport}-visible Will show the Element on the specified viewport only

The original Bootstrap {viewport}-visible classes, apply a default display: none property on all affected Elements, and would later override it with a display: block property, when the viewport-size is appropriate. This required a workaround for Elements which had a default display property other than block, usualy in the form of a {vieport}-display-{value} class.

The current {viewport}-visible classes surpass this issue, by not applying the display: none property as a default, but by adding it when the viewport-size is appropriate, and the Element does not have a {viewport}-visible class appropariate to that viewport.

* Using these, do not require additional fixes for elements which don't have display: block as default
* These will not override the hidden class, even if the viewport is appropriate (with proper usage, should not be an issue).
