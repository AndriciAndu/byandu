Appear Effects

Effects : Appear

I. Description

A collection of effects that trigger an 'appear' transition, when elements are scrolled into view.

* Effects can be triggered/applied-to individual elements or they can be sequenced through a parent element.

* Effects / sequences can be: auto-triggered when scroll-into-view or they can be triggered by custom functions.

* Sequences can have: normal or random trigger-order defined

* Sequences can have: normal or sequenced trigger-delays defined

* Sequences can be used for: full-page image galleries || bottom-page image-loading effects.

II. Initialization

Step 1. Download and refference the asterisk-effects-appear.css CSS file.

<link href="--myPath--/asterisk-effects-appear.css" type="text/css" rel="stylesheet">

Alternatively, a {*}-spec.css file can be generated within the [Code] Tab

<link href="--myPath--/asterisk-effects-appear-spec.css" type="text/css" rel="stylesheet">

Step 2. Download and refference the asterisk-effects-appear.js JS file.

<script src="--myPath--/asterisk-effects-appear.js" type="text/javascript"></script>

Step 3. Make the next function call :

asterisk.effects.appear.init() // if dynamic content - call after content has loaded

III. Usage

Adding an appear-effect on a HTML Element will result in :

1. The Element will be hidden from view, through the hasAppearEffect class.

2. The Element will be given a personal trigger, determined by it's position within the Document.

3. When the page is scrolled, and the Element comes into view, it will be made visible through the appearEffects-apply class.

The effect with which it will be made visible, is determined by the appear-{effect} class assigned to it.

Step 1. Add the static hasAppearEffect class on the HTML Element

Step 2. Add a appear-{effect} class on the HTML Element

ex : <div class="hasAppearEffect appear-slideIn-left"> </div>

ex : <div class="hasAppearEffect appear-fadeIn-btm"> </div>

ex : <div class="hasAppearEffect appear-scaleIn"> </div>

Step 3. Certain Effects require an with a effectsContainer-perspective class.

* Required for : spinIn-{*} , flyIn-{*} , landIn-{*} , fallIn-{*} effects.

* each such Element requires its' own effectsContainer-perspective Parent.

<div class="effectsContainer-perspective"> <div class="hasAppearEffect appear-flyIn-fwd-top"> </div> </div>

For best results, an should be created for each affected HTML Element, to hold the required effectsContainer-perspective class.

NOT Recommended : multiple elements within the same effectsContainer-perspective

<div class="effectsContainer-perspective">
<div class="hasAppearEffect appear-fallIn-fwd-left"> </div> <div class="hasAppearEffect appear-fallIn-fwd-left"> </div> <div class="hasAppearEffect appear-fallIn-fwd-left"> </div> ...
</div>

Recommended : each element within it's own effectsContainer-perspective

<div class="effectsContainer-perspective"> <div class="hasAppearEffect appear-fallIn-fwd-left"> </div> </div> <div class="effectsContainer-perspective"> <div class="hasAppearEffect appear-fallIn-fwd-left"> </div> </div> <div class="effectsContainer-perspective"> <div class="hasAppearEffect appear-fallIn-fwd-left"> </div> </div> ...

Synchronizing multiple HTML Elements will result in :

1. The Elements will (still) be hidden from view, through their hasAppearEffect class.

2. The Elements will not be given personal triggers. The trigger will be created for the effectsContainer-sequence Parent Element and it's position within the Document.

3. When the page is scrolled, and the effectsContainer-sequence Parent Element comes into view, it will make all the Elements within the sequence visible, by applying the appearEffects-apply class on each of them.

Creating a sequence requires placing the hasAppearEffect HTML Elements within a effectsContainer-sequence Parent.

* Even if triggered through a sequence, individual Elements can still have different appear-{effect} and individual delay values.

Effects which do not require effectsContainer-perspective :
<div class="effectsContainer-sequence"> <div class="hasAppearEffect appear-slideIn-left"> </div> <div class="hasAppearEffect appear-slideIn-top"> </div> <div class="hasAppearEffect appear-slideIn-right"> </div> </div>
Effects which do require effectsContainer-perspective :
<div class="effectsContainer-sequence">
<div class="effectsContainer-perspective"> <div class="hasAppearEffect appear-fallIn-fwd-left"> </div> </div> <div class="effectsContainer-perspective"> <div class="hasAppearEffect appear-fallIn-fwd-top"> </div> </div> <div class="effectsContainer-perspective"> <div class="hasAppearEffect appear-fallIn-fwd-right"> </div> </div>
</div>

Creating a custom trigger will result in :

1. The Elements will be hidden from view, through the hasAppearEffect class.

2. The Element / sequence container-element will not be given a personal trigger.

3. The Element / sequence container-element will not be made visibile when it comes into view.

4. The Element / sequence container-element can be made visible through a custom function, which adds the appearEffects-apply class.

Step 1. Add an appear-customTrigger class on each target.

* For elements : the class is added to the hasAppearEffect HTML Element itself.

<div class="hasAppearEffect appear-slideIn-left appear-customTrigger"> </div>
<div class="effectsContainer-perspective"> <div class="hasAppearEffect appear-flyIn-fwd-top appear-customTrigger"> </div> </div>

* For sequences : the class is added to the effectsContainer-sequence Parent Element.

<div class="effectsContainer-sequence appear-customTrigger">
<div class="hasAppearEffect appear-slideIn-left"> </div>
<div class="hasAppearEffect appear-slideIn-left"> </div>
<div class="hasAppearEffect appear-slideIn-left"> </div>
</div>
<div class="effectsContainer-sequence appear-customTrigger">
<div class="effectsContainer-perspective"> <div class="hasAppearEffect appear-flyIn-fwd-top"> </div> </div> <div class="effectsContainer-perspective"> <div class="hasAppearEffect appear-flyIn-fwd-top"> </div> </div> <div class="effectsContainer-perspective"> <div class="hasAppearEffect appear-flyIn-fwd-top"> </div> </div>
</div>

Step 2. Call the asterisk.effects.appear.target_applyEffect( targetElem ) function.

* For elements : the argument is the desired hasAppearEffect HTML Element to trigger.

<div id="myElem" class="hasAppearEffect appear-slideIn-left appear-customTrigger"> </div>
var myElem = document.getElementById('myElem'); function myCustomFunction() { // do something asterisk.effects.appear.target_applyEffect( myElem ) // this will make the Element visible };
* For individual HTML Elements, the asterisk.effects.appear.target_applyEffect function simply adds the appearEffects-apply class on the hasAppearEffect HTML Element passed as the argument.

* For sequences : the argument is the desired effectsContainer-sequence Parent Element to trigger.

<div id="mySequence" class="effectsContainer-sequence appear-customTrigger">
<div class="hasAppearEffect appear-slideIn-left"> </div>
<div class="hasAppearEffect appear-slideIn-left"> </div>
<div class="hasAppearEffect appear-slideIn-left"> </div>
</div>
var mySequence = document.getElementById('mySequence'); function myCustomFunction() { // do something asterisk.effects.appear.target_applyEffect( mySequence ) // this will make the Elements in the sequence visible };
* For sequences, the asterisk.effects.appear.target_applyEffect function : - itterates through all the hasAppearEffect HTML Elements within the sequence that is passed as the argument - adds the appearEffects-apply class on each HTML Element - includes the functionality for the delay and drandomOrder scenarios.

Adding a delay will result in :

When the Element / sequence reaches its' trigger point, the transition will be posponed for a period of time (the delay) before it takes effect.

* For Elements :

Option (a) : Add a transition-delay CSS attribute on the hasAppearEffect HTML Element.

<div id="myElem" class="hasAppearEffect appear-slideIn-left"> </div>
#myElem { transition-delay: 200ms; }

Option (b) : Add a transition-delay-{number} class on the hasAppearEffect HTML Element.

<div class="hasAppearEffect appear-slideIn-left transition-delay-200"> </div>
.transition-delay-200 { transition-delay: 200ms; }
* The transition-delay-{number} classes are included in the asterisk-effects-appear.css file. * The {number} ranges from 100 to 5000, in increments of 100.

* Both options cannot be used together - as the transition-delay CSS declarations would override one or the other

* For sequences :

Option (a) : Add individual delays on each desired hasAppearEffect HTML Element (steps from above)

Option (b) : Add a data-appearEffects-delay attribute on the effectsContainer-sequence Parent Element.

If the data-appearEffects-delay attribute is present :

1. It will automatically be picked up by Javascript

2. It will call the asterisk.effects.appear.target_applyEffect() for each hasAppearEffect Element, through a setTimeout

3. The setTimeout value is incremented for each Element

<div class="effectsContainer-sequence" data-appearEffects-delay="200"> <div class="hasAppearEffect appear-slideIn-left"> </div> <!-- will appear after 0ms --> <div class="hasAppearEffect appear-slideIn-left"> </div> <!-- will appear after 200ms --> <div class="hasAppearEffect appear-slideIn-left"> </div> <!-- will appear after 400ms --> <div class="hasAppearEffect appear-slideIn-left"> </div> <!-- will appear after 600ms --> </div>

* Both options cannot be used together - as the transition-delay CSS declarations and the setTimeout would both take effect

VI. 'Build-in' Synchronization Effect (HTML)

Create random-order synchronised sections (usual use case: sections of images)

<div class="effectsContainer-sequence" data-appearEffects-delay="100" data-appearEffects-randomOrder="true">
<div class="hasAppearEffect appear-slideIn-left"></div> <div class="hasAppearEffect appear-slideIn-left"></div> <div class="hasAppearEffect appear-slideIn-left"></div> <div class="hasAppearEffect appear-slideIn-left"></div> <div class="hasAppearEffect appear-slideIn-left"></div> <div class="hasAppearEffect appear-slideIn-left"></div> <div class="hasAppearEffect appear-slideIn-left"></div> <div class="hasAppearEffect appear-slideIn-left"></div> <div class="hasAppearEffect appear-slideIn-left"></div>
</div>
