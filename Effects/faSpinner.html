Component : faSpinner (FontAwesome Spinner)

Effects : faSpinner
Size
Speed

Collection of used to apply a loading-effect overlay to the Element.

Effect is achieved by using the FontAwesome library and CSS pseudo-elements.

The effect can be controlled by adding/removing the faSpinner class on the desired element.

Can be used for search queries, fetch / GET / POST requests, etc.

Step 1. Refference the FontAwesome Library.

<link rel="stylesheet" href="https://maxcdn.bootstrapcdn.com/font-awesome/4.7.0/css/font-awesome.min.css">

Step 2. Download and refference the asterisk-component-faSpinner.css CSS file.

<link href="--myPath--/asterisk-component-faSpinner.css" type="text/css" rel="stylesheet">

Alternatively, a {*}-spec.css file can be generated within the [Code] Tab

<link href="--myPath--/asterisk-component-faSpinner-spec.css" type="text/css" rel="stylesheet">

The effect is controlled by adding/removing the fa-spinner class on the target Element .

As the common use cases revolve around applying the effect while the HTML Element is updating it's content, toggling the faSpinner class is ussualy included within a larger function (the function which actually loads the content). 

															    function myFunctionToLoadContent() { 
															        myElem.classList.add('faSpinner');
															         // add some code to load content to myElem 
															        setTimeout(
															            function(){
															                myElem.classList.remove('faSpinner')
															            }, 0);
															    }

The setTimeout is required in order to display the effect, which is done by using two different tasks:

(1) on one task it adds the class, which renders the effect
(2) on the next task it remove the class and the effect

Without the setTimeout, the class would be added and removed on the same task, and the effect will not be visible.

More on the Event Loop and Tasks : Youtube link

The faSpinner class should be statically applied on the body, and removed through a function after the document has loaded.

<body class="faSpinner"> ... </body> // placed on the body - removed when document has loaded
JQuery 
														    $( document ).ready( 

														        $('body').removeClass("faSpinner")

														    );
Javascript 
														    document.onreadystatechange = function () { 
														        if (document.readyState === "interactive") {
														            document.body.classList.remove("faSpinner")
														        }
														    };

* The HTML Element to which the faSpinner class is applied to, must not have additional rules set on the :before or :after CSS pseudo-elements

* as these will interfere with the faSpinner's position / style / animation.

If the HTML Element does require CSS pseudo-elements for other purposes, it is recommended to use a to hold the faSpinner class.

<div class="faSpinner">
<div class="myElem"> ... </div>
</div>

The spinner is positioned in the center of the HTML Element to which the faSpinner class is applied to. While the spinner is animating, the HTML Element receives several styles - to not offset the display of the spinner.

* these style have the !important declaration, to override more-specific declarations, that might have been applied to the HTML Element.

* these will not affect the HTML Element when it is not animating the spinner

.faSpinner {
position: relative; padding: 0 !important; max-width: 100vw !important; max-height: 100vh !important; overflow: hidden !important;
}

The aditional classes must be added alongisde the faSpinner class, in order to modify specific styles.
