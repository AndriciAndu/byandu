Skeleton Overlay

Effects : Skeleton Overlay

Content placeholders -- improve perceiveload time, while the actual content is loading.

Applies an overlay over the Element, while the image is loading.

It is removed once the image has loaded.

Download and refference the asterisk-effects-ribbon.css CSS file.

<link href="--myPath--/asterisk-effects-ribbon.css" type="text/css" rel="stylesheet">

Alternatively, a {*}-spec.css file can be generated within the [Code] Tab

<link href="--myPath--/asterisk-effects-ribbon-spec.css" type="text/css" rel="stylesheet">
Types :

Step 1. Add the skeletonOverlay to the Element.

Step 2. Add the data-img-src data-attribute to the Element, with it's value set to the src value of the Image in question.

<div id="myElem" class="skeletonOverlay" data-img-src="myImageSrc"></div>

Step 3. Call asterisk.functionality.skeletonOverlay.CEVA(), with the Element passed as an argument.

var myElem = document.getElementById('myElem'); asterisk.functionality.skeletonOverlay.CEVA( myElem );

Once calling asterisk.functionality.skeletonOverlay.CEVA( myElem );, an image will be created, with the specified data-img-src src.

The created image will get an load Event attached to it.

Once the image has loaded, the load Event will set the remove the skeletonOverlay Class

'Single-Curl' Ribbons require a single <div> Element, with the appropriate ribbon--{connectingSide}--{curlPosition} Class.

They make use of the Element's ::before & ::after pseudo-elements.

The ribbon uses position: absolute, and is positioned in relation to it's nearest non-static-position Parent.

It can be set to follow the flow of the document (to be used as a Section Title, for example), by setting position: relative, and then using top / bottom / left / right or transform: translate to reposition.

This might require also adjusting the width property, if the Parent Element has paddings.

																		        <p> Paragraph lorem ipsum </p>
																		        <p> Paragraph lorem ipsum </p>
																		        <p> Paragraph lorem ipsum </p>
																		    </div>

'Multi-Curl' Ribbons can be adapted to the style of a 'Single-Curl', while also allowing for the presence of 'double-curls' on the Ribbon's connecting side.

This is unable with 'Single-Curl' Ribbons, because each curl requires the use of two Elements, and in 'Single-Curl' we can only use the two pseudo-elements available - and are able to only create one Curl.

This requires adding the ribbon--singleSide--{connectingSide} Class on the Ribbon Element.

																	    <div class="ribbon--horizontal ribbon--singleSide--left">
																	        <div class="ribbon__curl ribbon__curl--left--top"></div>
																	        <div class="ribbon__curl ribbon__curl--left--btm"></div>
																	        <div class="ribbon__text"> Ribbon Text </div>
																	    </div>
																	    <div class="ribbon--vertical ribbon--singleSide--top">
																	        <div class="ribbon__curl ribbon__curl--top--left"></div>
																	        <div class="ribbon__curl ribbon__curl--top--right"></div>
																	        <div class="ribbon__text"> Ribbon Text </div>
																	    </div>

mix-blend-mode is inneficient.

animations are expensive - even animating 'ideal' properties such as transforms and opacity, still take their toll on the user's resources.

As is logical - having many keyframes - modifying multiple properties - running multiple animations at once - each of these have their cost. If managed properly, they might be truly unoticeable - but they must be considered and approached adequately.

The pros :

No extra elements required - CSS only solution.

No reflows ? - the markup itself is not changed

The neutral :

CSS animations - would have been necessary with any approach

The cons :

Not self-adaptable - as we are using font-icons, we cannot set their dimensions through the width property - we can only use the font-size property - this means: all icons must have the same size - the elements themselves cannot adapt to the size of the container.

Yet, they don't have to fit perfectly - all major sitew that used skeleton screens, have them as a generic novalty instead of a set dimension. And, as their purpose is to suggest to the user that the content is loading, and they will be shown only briefly, it makes little difference in the long run.
