Baseline Tabs component, with CSS-only functionality (no JS required for it to work).

* CSS-only - will function properly if the user has JS disabled.

* No need for unique name values for each Component across the document.

* No need for unique id values for linking each Button and Label.

* Can nest multiple cssTabs within each other, without any issue.

Download and refference the asterisk-component-cssTabs.css CSS file.

<link href="--myPath--/asterisk-component-cssTabs.css" type="text/css" rel="stylesheet">

Just add the Component's Markup :

* cssTabs-btn, cssTabs-btnLabel & cssTabs-content elements are synchronised with each other by having the same data-tabs-id value.

* each synchronised cssTabs-btn, cssTabs-btnLabel and cssTabs-content elements require an unique data-tabs-id value, within the cssTabs. 

												    <form class="cssTabs">
												        <input class="cssTabs-btn" name="cssTabs-btn" type="radio" data-tabs-id="1" checked="checked"/>
												        <input class="cssTabs-btn" name="cssTabs-btn" type="radio" data-tabs-id="2"/>

												        <div class="cssTabs-labelGroup">
												            <div class="cssTabs-btnLabel" data-tabs-id="1"> Btn-Text 1 </div>
												            <div class="cssTabs-btnLabel" data-tabs-id="2"> Btn-Text 2 </div>
												        </div>

												        <div class="cssTabs-contentGroup">
												            <div class="cssTabs-content" data-tabs-id="1"> Content 1 </div>
												            <div class="cssTabs-content" data-tabs-id="2"> Content 2 </div>
												        </div>
												    </form>

Note : The Elements' placement and order are very important, in order for the functionality to work properly :

1st : the cssTabs-btn Elements

2nd : the cssTabs-labelGroup Element, containing all cssTabs-btnLabel Elements

3rd : the cssTabs-contentGroup Element, containing all cssTabs-content Elements

Adding extra tabs/sections :

Step 1. Add a new input (type="radio") with the cssTabs-btn class, within the cssTabs Element.

Step 2. Add a new div with the cssTabs-btnLabel class, within the cssTabs-labelGroup Element.

Step 3. Add a new div with the cssTabs-content class, within the cssTabs-contentGroup Element.

Step 4. Give the newly-added cssTabs-btn, cssTabs-btnLabel and cssTabs-content Elements the same data-tabs-id value.

* the data-tabs-id value must be unique within the current cssTabs Parent Element. 

															    <form class="cssTabs">
															        <input class="cssTabs-btn" name="cssTabs-btn" type="radio" data-tabs-id="1" checked="checked"/>
															        <input class="cssTabs-btn" name="cssTabs-btn" type="radio" data-tabs-id="2"/>
															         <input class="cssTabs-btn" name="cssTabs-btn" type="radio" data-tabs-id="n"/> 

															        <div class="cssTabs-labelGroup">
															            <div class="cssTabs-btnLabel" data-tabs-id="1"> Btn-Text 1 </div>
															            <div class="cssTabs-btnLabel" data-tabs-id="2"> Btn-Text 2 </div>
															             <div class="cssTabs-btnLabel" data-tabs-id="n"> Btn-Text n </div> 
															        </div>

															        <div class="cssTabs-contentGroup">
															            <div class="cssTabs-content" data-tabs-id="1"> Content 1 </div>
															            <div class="cssTabs-content" data-tabs-id="2"> Content 2 </div>
															             <div class="cssTabs-content" data-tabs-id="n"> Content n </div> 
															        </div>
															    </form>

Placing the markup within a <form> Element, scopes the radio checkboxes (cssTabs-btn) within the parent.

doesn't require unique name values, for each individual cssTabs component.

* can have margins / paddings / borders.
