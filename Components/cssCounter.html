I. Description

A simple CSS-only Animated Counter. (CSS animation, Javascript animation trigger)

This is a simple, ongoing exercise, because the internet refused to provide a decent solution for this topic, or it mentioned the blasphemy that it can't be done.

Just because you could, doesn't mean you should use this approach.

* It is still a markup-heavy and static implementation

* It would be much more dynamic and efficient, to simply use Javascript.

II. Initialization

Download and refference the asterisk-component-cssCounter.css CSS file.

<link href="--myPath--/asterisk-component-cssCounter.css" type="text/css" rel="stylesheet">

This contains only the baseline styles for the CSS Counter, without the animations themselves, as each case requires different HTML Markup and animation properties.

II. Usage

To generate the cssCounter Element's HTML Markup and the CSS animation code, use the Code Generator within the Code Tab

* The top-screen menu, right button.

Step 1. Add the generated HTML to your Document - defining the cssCounter-run Element.

Step 2. Add the generated CSS to your Document - defining the cssCounter-run Element's animation.

Step 3. Add the cssCounter-run class on the cssCounter Element, when the animation should take effect.

III. Markup Roles

The cssCounter Element acts as the Component's visible interface. For this, it requires to have :

1. width: auto and display: inline-block

2. a set height value, equal to an cssCounter-item's height value

3. overflow: hidden

The cssCounter-container is a wrapper for the cssCounter-item Elements. It will receive the actual animation.

It must have the default height: auto to expand-to-fit all cssCounter-item Child Elements.

The effect is created by using transform: translateY() within an animation, applied to the cssCounter-container Element.

There needs to be a number of cssCounter-item Elements, equal to the number we want to count to (count to 10 => 10 cssCounter-item Elements)

The counter-increment: x and the ::after content: counter(x), are used to generate the numbers for each Item

The outline property is used instead of border to not affect the flex alignment.

If one prefferes using the border property, to properly align the numbers, it requires setting a line-height value equal to the height minus the top and bottom border-width.
