Client-Side Routing

Functionality : Route (Client Side)

Baseline Client-Side Routing System, for simulating SPA / AJAX-based functionality.

* Updates parts of an application, without triggering a page reload

* No hash routing required ( None of this : myDomain/#/myPage.html )

* Supports navigating to a page directly ( ex : mySite/something/etc.html )

* currently supports only static paths

Within a Tradition (multi-page) Website/Application, each individual Page is 'self-sustained', having it's own DOCTYPE Declaration, Meta Tags, CSS/JS Files, HTML markup, etc.

When navigating to different pages, the browser discards the current page completely, requests new files from the server, and loads a completely new Document.

Within a Single-Page Application (SPA) environment, there is a Root Page which is 'self-sustained', just like any Page within a Tradition (multi-page) Website/App.

* The Root Page is ussualy the index.html Page.

Navigating to different pages happens by updating part of the Root Page (the part that is updated is called a View), and this happens without actually navigating away from the Application's Root Page * Hence the name of 'single-page'

* The Root Page is basically the index.html Page. It is created like any (multi-page) website's Page, and has the structure of a generic stand-alone page.

Step 1. Create a normal, stand-alone index.html File.

Step 2. Within the Root Page, add the <div id="__route__mainView"></div> (representing the View - where each individual Page's content will be loaded)

Step 3. At the bottom of the Root Page, just before the closing </body> Tag, add the asterisk-route.js JS File. 

															    // Root Page Example 

															    <!DOCTYPE html>
															    <html>
															      <head>
															          // all the application's Meta Tags 
															          // all general-purpose CSS files 
															      </head>
															      <body>
															          // other HTML content 
															          <div id="__route__mainView"> </div>    // this is the Main View, where each Page's content is loaded 
															          // other HTML content 
															          // all general-purpose JS files
															          <script src="<--myPath--/asterisk-route.js"></script>
															      </body>
															    </html>

Each 'individual' Page consists of a .html File, the content of which will be placed within the View, whenever that Page is loaded.

Step 1. Create the myPage.html HTML File.

Step 2. Refference the asterisk-route-intermediary.js JS File, at the top of markup.

Step 3. Create an entry within the asterisk.route.paths 

															    // 'Individual' Page Example 

															    <script src="--myPath--/route_intermediary.js"></script> // * Required 
															    <style> /* Add page-specific styles */ </style>          // * Optional
															    <article>
															        // Place content here 
															    </article>

* Each individual Page does not require refferencing to the CSS/JS Files which are already loaded on the Root Page.

* Specific styles can be loaded for the individual Page, within the <style></style> Tag at the top of the HTML File.

* Specific scripts can be loaded for each individual Page, by creating a myArticle.js File and refferencing it within the asterisk.route.paths.

As we never navigate away from the Root Page (even though the url changes), to load a new 'individual' Page within the View, we need to define what files will be used in this process.

For this, we must create a Path Object for each 'individual' Page within our application. These are stored within the asterisk.route.paths Array.

Each Path Object is an unnamed Object, with 3 Properties :

Property Mandatory Type Description
filePath_html Yes String The path to the Page's .html File
filePath_js No String / Null The path to the Page's .js File
articleParameters No Object Stores any parameters required to bring additional changes to Page / Application

Within this website (byandu.netlify.com), each Path Object has a category property, which is used to update the Category Buttons within the left-side navigation menu. 

														    asterisk.route.paths = [
														        { 
														            filePath_html : '/myCategory/myArticle_1.html' , 
														            filePath_js   : '/js/myCategory/myArticle_1.js' , 
														            articleParameters : {
														                category : 'myCategory'
														            }
														        } ,

														        { 
														            filePath_html : '/myOtherCategory/myArticle_2.html' , 
														            filePath_js   : null , 
														            articleParameters : {
														                category : 'myOtherCategory'
														            }
														        } 
														    ];

There are 4 Intermediary Functions that can be used to control what happens when a new Page is loaded. They are optional.

These are already defined as empty functions, within the asterisk-route.js JS File, to avoid Refference Errors.

1. asterisk.route.intermediary.loadPage_onBegin()

This runs immediately when a new Page starts loading. It can be used to add a loading animation on the View, while the content is updating.

Example : In the current website (byandu.netlify.com) it is used to add a faSpinner loading animation, to the View. 

															    asterisk.route.intermediary.loadPage_onBegin = function() {
															        asterisk.route.info.mainView.classList.add('faSpinner', 'faSpinner-fast', 'faSpinner-lg');
															    };

2. asterisk.route.intermediary.loadPage_updateDOM()

This is used to bring changes to the Elements which are outside the View, when loading a new Page.

Example : In the current website (byandu.netlify.com) it is used to update the active status on the categories/articles buttons within the left side navbar.

* In the example, the articleParams__obj reffers to the current Path Object, defined within asterisk.route.paths. 

															    asterisk.route.intermediary.updateDOM = function(articleParams__obj) {

															        var myTargets_1 = Array.from(document.getElementsByClassName('__route__category'));
															        var identifierValue = articleParams__obj.articleParameters.category;

															        myTargets_1.map(function(elem) {
															            if (elem.getAttribute('data-categoryName') == identifierValue) { 
															                elem.classList.add('active') 
															            } else { 
															                elem.classList.remove('active') 
															            }
															        });

															        var myTargets_2 = Array.from(document.getElementsByClassName('__route__articleBtn'));
															        var identifierValue = articleParams__obj.filePath_html;

															        myTargets_2.map(function(elem) {
															            if (elem.pathname == identifierValue) { 
															                elem.classList.add   ('active') 
															            } else { 
															                elem.classList.remove('active') 
															            }
															        });
															    };

3. asterisk.route.intermediary.loadPage_runDefaultScript()

This is used as a default script, for the content within the View, after a new Page has loaded. Checks/functions which should occur for every new Page load, should be included here to avoid repeating them across each Page's .js File.

Example : In the current website (byandu.netlify.com) it is used to do some formatting on the text within <pre> Elements. 

															    asterisk.route.intermediary.loadPage_runDefaultScript = function() {
															        var targetedPreElements = Array.from(document.getElementsByClassName('pre-removeTabSpaces'));

															        targetedPreElements.map(function(item) {
															            item.innerHTML = string_remove_tabSpaces(item.innerHTML);
															        });
															    };

4. asterisk.route.intermediary.loadPage_onEnd()

This is the last thing that runs when loading a new Page (after the Page's content is generated, and the Page's Javascript has executed). It can be used to remove the loading animation on the View, to check which Elements were removed from the DOM (memory management), etc.

Example : In the current website (byandu.netlify.com) it is used to remove the faSpinner loading animation. 

															    asterisk.route.intermediary.loadPage_onEnd = function() {
															        asterisk.route.info.mainView.classList.remove('faSpinner', 'faSpinner-fast', 'faSpinner-lg');
															    };

When navigating directly to the Root/Index Page : (ex: byandu.netlify.com )

The 'scheleton' markup is parsed and generated

The general-use CSS/JS Files are parsed and memorized

The asterisk-route.js file is parsed and a 'default' page is generated within the View

From here, navigating to a new page will simply update the View with the coresponding data.

When navigating directly to a specific Page : (ex: byandu.netlify.com/effects/appear.html )

The 'scheleton' markup is not present within the appear.html File - as it contains only the HTML for the article itself

The general-use CSS/JS Files are not refferenced

The asterisk-route.js file is not refferenced and it's functionality cannot take effect.

By itself, navigating to a specific path (without going to the Root/Index Page first), would leave the user with a basic HTML file
