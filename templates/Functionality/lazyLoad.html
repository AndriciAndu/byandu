Functionality : Lazy Loading

I. Description

Functionality which loads different-sized images, depending on the user's screen size.

II. Initialization

Step 1. Download and refference the [ asterisk-functionality-lazyLoad.js ] JS file.

<script src="--myPath--/asterisk-functionality-lazyLoad.js" type="text/javascript"></script>

Step 2. Make the next function call :

asterisk.functionality.lazyLoad.init();

III. Usage

Step 1. Add the lazy-load class on the <img> Element.

* Allows the <img> to be targeted through Javascript.

<img class="lazy-load"/>

Step 2. Define the src attribute on the <img> Element.

* This image will be loaded as default, before the lazy-load functionality takes effect.

Having a src defined is required to allocate the element's space within the document, to not have the images collapsed and then have the document snap into place when the images actually load.

1. An universal placeholder image

Pros:

------- * Fast allocation of space

* Only one HTTP Request required for each type of image (ratio / orientation)

* Only one template must be created for each type of image (ratio / orientation)

* An image-ratio / image-orietation parameters must be defined, to properly allocate the space for the<img> Element, within the Document.

Cons:

* It might offer a lesser user experience, until the actual image is loaded

2. A thumbnail version of the image

This represent a extremely small (pixelated) version of the actual image (ex: 120x90 px from 1200x900 px), with a blur filter applied to it. This is recommended for large background images, ussualy above-the-fold or section backgrounds

Pros:

* The image-ratio / image-orietation parameters are not required.

* Improved user experience in comparison to

Cons:

* Multiple HTTP Requests to load initial versions

This requires multiple http requests (one for each image), yet the file size is very small

3. The xs media type version of the image

<img class="lazy-load" src="myImg.jpg"/>

Step 3. Define the data-lazy-srcset attribute.

The data-lazy-scrset contains the scr values for the different media-type scenarios.

The data-lazy-scrset attribute must have the value as a JSON-format Object.

The data-lazy-scrset value can be split across multiple lines, as all line-breaks, tab-spaces and non-single-spaces are removed, before parsing the JSON value. (like in the example)

<img class="lazyLoad" src="../../images/myImage-thumb.jpg"
data-lazy-srcset='{
"xs"   :  "../../images/myImage-xs.jpg"   , "sm"  :  "../../images/myImage-sm.jpg"  , "md"  :  "../../images/myImage-md.jpg" , "lg"    :  "../../images/myImage-lg.jpg"   , "xl"    :  "../../images/myImage-xl.jpg"
}'/>

If certain media-type scenarios are to-be-skipped

<img class="lazyLoad" src="../../images/myImage-thumb.jpg"
data-lazy-srcset='{
"xs"   :  "../../images/myImage-xs.jpg"   , "sm"  :  "../../images/myImage-sm.jpg"  , "md"  :  "../../images/myImage-md.jpg" , "lg"    :  "../../images/myImage-lg.jpg"   , "xl"    :  "../../images/myImage-xl.jpg"
}'/>
