Better max-height transitions with multiple transition-delay

The most common use-case for collapsible components, such as accordion | spoiler | navbar | etc is to use a max-height transition, alongside overflow: hidden, to generate a collapse/expand animation, for the user to see.

The issue is that setting max-height: auto is invalid.

And so we are obligated to set a default value of huge proportions (max-height: 5000px or max-height: 10000px), to make sure that the potential height of the content is lower than the max-height provided.

The closer the max-height value to the actual height of the content, the better/smoother the transition effect will be, but we must always make sure that the max-height is larger.

.collapsibleElement { overflow: hidden; max-height: 0; transition: max-height 500ms; }
.collapsibleElement.expanded { max-height: auto; /* invalid */ }
.collapsibleElement.expanded { max-height: 5000px; }

This leads to scenarios where:

* the content is expanded at a speed close to what is desired

* yet the collapse seems to have a 'buffer', before the transition takes place.

Of course, this 'buffer' happens because of the difference between the max-height value, and the height of the content itself.

It does not appear to have a 'buffer' when it is expanding because it starts from 0 and it immediately engulfs the actual height of the content.
When it is collapsing, it starts from 5000px/10000px, and it takes a good portion of it's allocated time to get to the height of the actual content, before the effect is visible.
Let's consider:
Our max-height: 5000px ; Our actual content height: 1000px ; Our transition duration: 500ms ; Result => our speed: 1000px/100ms;
When expanding: First 100ms: our whole content will show Latter 400ms: apparently nothing is happening, as the height has already reached it's maximum - but actually the max-height is still 'growing' even though nothing is displayed. Conclusion: we have a 100ms transition, with a 400ms 'after-effect'
When collapsing: First 400ms: apprently nothing is happening, but the max-height is slowly counting down Latter 100ms: the max-height has reached the size of our actual height, and the transition begins, as it covers our content height. Conclusion: we have a 400ms delay, with a 100ms transition
