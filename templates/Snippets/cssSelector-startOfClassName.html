Part-of-className CSS Selector

Snippets : start-of-className Selector

CSS Selector Syntax, which targets elements which have a class that starts with a specific string. 

							    [class^="classNameString"] , [class*=" classNameString"] {
							        /* do magic */
							    }

Example - both scenarios achieve the same results : 

						    imgFilter-1977  , imgFilter-aden    , imgFilter-amaro    ,
						    imgFilter-ashby , imgFilter-brannan , imgFilter-brooklyn {
						        /* do something */
						    }
							


						    [class^="imgFilter-"] , [class*=" imgFilter-"] {
						        /* do something */
						    }

All HTML Attributes have String values.

CSS Attribute Selectors are basically regex verifiers (check if a string contains a substring).

The class Attribute is taken/checked as a single String, not as individual classes.

If we have a div like this : <div class="myClass   myOtherClass   not-myClass   myClass-qwerty"> </div>
Its' class Attribute is : "myClass   myOtherClass   not-myClass   myClass-qwerty"
It contains "myClass" : "myClass   myOtherClass   not-myClass   myClass-qwerty"
It also contains "ass not-myCl" : "myClass   myOtherClass   not-myClass   myClass-qwerty"

(1) [class^="myClass"] This selector checks if the class Attribute string begins with the given value

[attr^=value] Represents elements with an attribute name of attr whose value is prefixed (preceded) by value - MDN

<div class="myClass   myOtherClass   not-myClass   myClass-qwerty"> </div> - This will pass (at the very beginning of the String) <div class="myOtherClass   myClass   not-myClass   myClass-qwerty"> </div> - This will not pass (it has it, but not at the beginning)

(2) [class*=" myClass"] This selector checks if the class Attribute string contains the given value

[attr*=value] Represents elements with an attribute name of attr whose value contains at least one occurrence of value within the string - MDN

The syntax also includes a space character before the actual value - As all class names are separated by a space.

Important : [class*="myClass"] (no space character) is not the same as [class*=" myClass"] (with space character)

[class*="myClass"] <div class="myClass   myOtherClass   not-myClass   myClass-qwerty"> </div>
[class*=" myClass"] <div class="myClass   myOtherClass   not-myClass  myClass-qwerty"> </div>

As all classes are separated by a space, we use the (2) Syntax to target all classes which begin with the specified value.

[class*=" myClass"] <div class="myClass  myClass-2 not-myClass  myClass-3"> </div>

As the first class in the String doesn't have a space preceeding it, we use the (1) Syntax to check the beginning of the String.

[class^="myClass"] <div class="myClass myClass-2 not-myClass myClass-3"> </div>

Using both together :

[class^="myClass"] , [class*=" myClass"] <div class="myClass  myClass-2 not-myClass  myClass-3"> </div>
This is especially usefull for large groups of classes or libraries, where there are a lot of classes involved, and where the collection could have items removed/added along the way.
