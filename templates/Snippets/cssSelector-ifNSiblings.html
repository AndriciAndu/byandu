Snippets : if-n-siblings

CSS Selector Syntax, which targets elements where there are exactly n children/siblings.

Found from Lea Verou | CSS3 Secrets: 10 things you might not know about CSS3 | Fronteers 2011 .

Not Scoped - TagName Identifier 

							    div:first-child:nth-last-child(n),
							    div:first-child:nth-last-child(n) ~ div {
							        /* do magic */
							    }

Not Scoped - Class Identifier 

							    .myClass:first-child:nth-last-child(n),
							    .myClass:first-child:nth-last-child(n) ~ .myClass {
							        /* do magic */
							    }

Parent Scope - TagName Identifier 

							    .myParentClass div:first-child:nth-last-child(n),
							    .myParentClass div:first-child:nth-last-child(n) ~ div {
							        /* do magic */
							    }

Parent Scope - className Identifier 

							    .myParentClass .myClass:first-child:nth-last-child(n),
							    .myParentClass .myClass:first-child:nth-last-child(n) ~ .myClass {
							        /* do magic */
							    }

Generic Example 

							    div:only-child {
							        width: 100%;
							    } 

							    div:first-child:nth-last-child(2),
							    div:first-child:nth-last-child(2) ~ div {
							        width: 50%;
							    } 

							    div:first-child:nth-last-child(3),
							    div:first-child:nth-last-child(3) ~ div {
							        width: 33.3%;
							    } 

							    div:first-child:nth-last-child(4),
							    div:first-child:nth-last-child(4) ~ div {
							        width: 25%;
							    }

(1) div:first-child:nth-last-child(n) This selector targets only the 1st Child Element itself

* If there are 2 child elements, the 1st child is also the 2nd-to-last child div:first-child:nth-last-child(2)

* If there are 3 child elements, the 1st child is also the 3rd-to-last child div:first-child:nth-last-child(3)

* If there are 4 child elements, the 1st child is also the 4th-to-last child div:first-child:nth-last-child(4)

* and so on ..

(2) div:first-child:nth-last-child(n) ~ div This selector targets all div elements, which come after the 1st Child Element

If there are exactly n children, target all div elements that follow it div:first-child:nth-last-child(n) ~ div

The general sibling combinator (~) separates two selectors and matches the second element only if it follows the first element (though not necessarily immediately), and both are children of the same parent element. (source : MDN)

Linking the two rules, it will target both the 1st Child Element and all div elements that follow it.

Setting n

The n placeholder should represent the exact number of siblings.

identifier:first-child:nth-last-child(n), identifier:first-child:nth-last-child(n) ~ identifier { /* do magic */ }

Example: If n = 3 , then the selector will target all identifier elements, where there are at exactly 3 siblings.

div:first-child:nth-last-child(3), div:first-child:nth-last-child(3) ~ div { /* do magic */ }
Case: 2 siblings div elements will not be targeted
<div> <div> </div> <div> </div> </div>
Case: 3 siblings div elements will be targeted
<div> <div> </div> <div> </div> <div> </div> </div>
Case: 4 siblings div elements will not be targeted
<div> <div> </div> <div> </div> <div> </div> <div> </div> </div>

All siblings will be counted, regardless of their tagName, or if they have the specified Identifier or not.

Case: 2 siblings div elements will not be targeted
<div> <div> </div> <input/> </div>
Case: 3 siblings div elements will be targeted
<div> <div> </div> <img> <p> </p> </div>
Case: 4 siblings div elements will not be targeted
<div> <form> </form> <div> </div> <div> </div> <img/> </div>
Setting Identifier

The Identifier can be any type of Selector.

Example : Identifier is set to .myClass
.myClass:first-child:nth-last-child(3), .myClass:first-child:nth-last-child(3) ~ .myClass { /* do magic */ }
This will style all elements with the myClass class, if there are exactly 3 siblings.

Initial conditions do not pass - nothing will be targeted

Scoping

The rules can be scoped to particular elements, by following the normal CSS Selector chaining logic.

.myParentClass .myClass:first-child:nth-last-child(n), .myParentClass .myClass:first-child:nth-last-child(n) ~ .myClass { /* do magic */ }
This will style all elements which are children of .myParentClass, which have the myClass class, if there are exactly n siblings.
Overriding values

To set adequate values for incrementing numbers of siblings, simply cascade the rules, in order :

*identifier*:first-child:nth-last-child(2), *identifier*:first-child:nth-last-child(2) ~ *identifier* { /* do magic */ } *identifier*:first-child:nth-last-child(3), *identifier*:first-child:nth-last-child(3) ~ *identifier* { /* do magic */ } *identifier*:first-child:nth-last-child(4), *identifier*:first-child:nth-last-child(4) ~ *identifier* { /* do magic */ } etc ...
