The 'Loading Spinner' Component

Components : loadingSpinner

I. Description

Simple loading-spinner, used as an screen overlay, while the website is loading.

* Once the document is ready, the loading-spinner is removed.

II. Initialization

Step 1. Add the CSS.

Option a) Download and refference the [ asterisk-component-loadingSpinner.css ] CSS file.

<link href="--myPath--/asterisk-component-loadingSpinner.css" type="text/css" rel="stylesheet">

Option b) Download and refference the [ asterisk-component-loadingSpinner-spec.css ] CSS file.

<link href="--myPath--/asterisk-component-loadingSpinner-spec.css" type="text/css" rel="stylesheet">

Option c) Include the component-code generated in the [Demo] tab or [Code] [CSS-singular]

<link href="--myPath--/asterisk-component-loadingSpinner-spec.css" type="text/css" rel="stylesheet">

III. Usage

Step 1. Add the Component's markup to your HTML.

* Recommended to be placed as the first Element within the body, right after the <body> tag.

Step 2. Change the loader-{type} class to apply the desired animation type.

<div class="websiteLoader loader-bars">
<div class="websiteLoader_container">
<div class="loader_element loader_element1"> </div> <div class="loader_element loader_element2"> </div> <div class="loader_element loader_element3"> </div> <div class="loader_element loader_element4"> </div>
</div>
</div>

Step 3. Add the next JS snippet.

JQuery
  $( document ).ready( $('body').removeClass("faSpinner") );  
Javascript
document.onreadystatechange = function () {
if (document.readyState === "interactive") { document.body.classList.remove("faSpinner") }
};

IV. Markup Roles

* allocates the available space for the entire box-model of the component

* centers the content (horizontal and vertical).

* allocates the size of the actual loader/loader_elements.

* changing it's size, will update the size of the actual loader/loader_elements.

The loader_element elements are individual parts that make up the actual loader.

Not every loader requires all 4 of the loader_element elements, yet it is recommended to keep the full markup to easily switch between loader-styles.

(when applying a 'loader-style class' it automatically sets the unused loader_element elements to display: none).

V. Styles

Changing the spinner type requires just changing the loader-*style* class on the websiteLoader Element.
<div class="websiteLoader loader-yinyang"> ... </div>
Changing the spinner speed requires just adding the loader-speed-* class on the websiteLoader Element.
<div class="websiteLoader loader-bars loader-speed-fast"> ... </div>

VI. Styling

Size | Color : Changing the values in the CSS file, in the shared styling of [ .websiteLoader ] :
.websiteLoader { ... --spinner-root-size: 100px; --spinner-root-color: #FFFFFF; --spinner-root-borderWidth: 5px; }
  Affected Element .websiteLoader_container .loader_element .loader_element
  Affected Style width | height bgColor | borderColor size of certain elements
  Effect size of actual spinner element color size of certain elements
Changing any of these values will update *most spinner-types, accordingly. *Certain spinner-types have more-specific variables defined. Example: .loader-circular-simple { --root-borderWidth: 15px; }
Speed | Delay : Each type of Spinner also has 2 localized variables, for controling the baseline animation speed/delay. Modifying these, will change the animation speed/delay for that type of spinner only.
Example:
.loader-bars { --spinner-root-animation-speed: 1000; --spinner-root-animation-delay: 100; }
.loader-bars.loader-speed-fast { --spinner-root-animation-speed: 750; --spinner-root-animation-delay: 75; }
.loader-bars.loader-speed-slow { --spinner-root-animation-speed: 1500; --spinner-root-animation-delay: 150; }
