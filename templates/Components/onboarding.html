The 'Onboarding' Component

Components : Onboarding
I. Initialization 1. Add the following files:
* [asterisk-component-onboarding.css],
* [asterisk-component-onboarding-stepsDefinitions.js],
* [asterisk-component-onboarding.js]. Required: The [ asterisk-component-onboarding-stepsDefinitions.js ] file, must be placed before the [ asterisk-component-onboarding.js ] file. 2. Make the next function call:
componentsObject.onboarding.init()
componentsObject.onboarding.init(current_target)
Case: componentsObject.onboarding.init() All [onboarding] within the document will be initialized.
Case: componentsObject.onboarding.init( [onboarding Element] ) That specific [onboarding] will be initialized.
Case: componentsObject.onboarding.init( [NON-onboarding Element] ) All [onboarding] within that [NON-onboarding Element] Element will be initialized.
II. Usage (HTML & JS) Required Markup :
<div class="onboarding"> <div class="onboarding-notification"> <div class="onboarding-notification-textBox"> </div> <div class="onboarding-notification-btnContainer"> <div class="onboarding-prevBtn"> Back </div> <div class="onboarding-nextBtn"> Next </div> </div> </div> <div class="onboarding-finishBtn"> Finish </div> </div>
// Full-screen container element // Notification Box - shows current scenario // Text to be displayed within Notification Box // Holds next/prev buttons and sets their size // Go to next step // Go to prev step     // Finish (hide) [onboarding]  
Steps Object :
Each [onboarding] requires setting up a [stepsObj] property, in which we define the actions which should occur on each step. This is done within [asterisk-component-onboarding-stepsDefinitions.js].
[onboarding].stepsObj = [ { // Step 1 - actions notifBox_setPosition : 'center-on-screen' , notifBox_setBubble : ' ' , notifBox_setText : '<p> First Step </p>' } , { // Step 2 - actions step_defaultTarget : document.getElementById('myId') , highlight : true , scrollTo : true , notifBox_setPosition : [['right-out', 10], ['center', 10]] , notifBox_setBubble : 'bubble-left bubble-center' , notifBox_setText : '<p> Second Step </p>' } , ... // Add more objects like above, to define more steps ];
The [stepsObj] property is an [array] of [annonimous objects]. Each [annonimous object], represents a [step] which the [onboarding] execution will follow. Each [step] can receive any of the following [properties], which will define what actions will occur :
[onboarding].stepsObj = [ { step_defaultTarget : click : highlight : scrollTo : notifBox_setPosition : notifBox_setBubble : notifBox_setText : notifBox_show : notifBox_hide : } ];
    Singular HTML Element - Default targetElement refference Singular/Multiple HTML Elements - will be 'clicked' Singular/Multiple HTML Elements - will be 'highlighted' Singular HTML Element - page will scroll to it Set the position of the notifBox in refference to the [step_defaultTarget] Set the bubble-position of the notifBox Set the innerHTML of the notifBox Display the notifBox (if it was hidden previously) Hide the notifBox
step_defaultTarget :
Property
Value
Description
step_defaultTarget
* HTMLElement
element to refference by default, if actions are set without a provided target.
Example: step_defaultTarget : document.getElementById('myId-1')
click :
Property
Value
Description
click
* true (boolean)
* HTMLElement
* array of HTMLElements
HTML Elements which will be click()-ed when this step runs.
Example:
click : true
// will click() the Element provided as [step_defaultTarget]
click : document.getElementById('myId-1')
// will click() the Element provided
click : [
document.getElementById('myId-1'),
document.getElementById('myId-2')
]
// will click() each of the Elements provided
Note : Order is important. If multiple Elements were provided as arguments (array, HTMLCollection), they will be targeted for the click() event, in the order they were defined within the array/classCollection. If certain scenarios require the click() events to be triggered in a specific order, set the order of the arguments within the array/classCollection accordingly.
highlight :
Property
Value
Description
highlight
* true (boolean)
* HTMLElement
* array of HTMLElements
HTML Elements which will be 'highlighted' (give the 'onboarding-highlight' class) when this step runs.
Example:
highlight : true
// will highlight the Element provided as [step_defaultTarget]
highlight : document.getElementById('myId-1')
// will highlight the Element provided
highlight : [
document.getElementById('myId-1'),
document.getElementById('myId-2')
]
// will highlight each of the Elements provided
Note : On each step, all elements which are already 'highlighted', will have their highlight removed.
To keep certain elements 'highlighted' throughout multiple steps, it is required to set the as targets for [highlight] in each step.
scrollTo :
Property
Value
Description
scrollTo
* true (boolean)
* HTMLElement
Element to which the page will scroll to (singular HTML Element).
Example:
scrollTo : true
// will scrollTo the Element provided as [step_defaultTarget]
scrollTo : document.getElementById('myId-1')
// will scrollTo the Element provided
Note : Only one Element can be defined as the target for scroll, as to not cause infinite looping, triggered by successively trying to reach multiple points throughout the Document. If multiple Elements are provided as arguments (array, HTMLCollection), only the first Element will be taken into account.
notifBox_setPosition :
Property
Value
Description
notifBox_setPosition
* 2D Array Construct
* 'center-on-screen'
Will define the position of the [onboarding-notification] box within the document - in refference to the [step_defaultTarget].
notifBox_setPosition : 'center-on-screen'
This scenario will center the [onboarding-notification] box, in relation to the [onboarding] component, and will appear as if fixed to the center of the viewport.
notifBox_setPosition : [ [ x-axis-refference , x-axis-offset-value ] , [ y-axis-refference , y-axis-offset-value ] ]
This scenario will position the [onboarding-notification] box, in relation to the [step_defaultTarget] DOM Element, in accordance with the provided parameters within the 2D Array Construct. (See 'Code Generator & Demo' below)
