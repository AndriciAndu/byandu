Hover Text

Component : Tooltip (responsive)

I. Description

A collection of utility classes, which add a text pseudo-element that appears when the element is hovered.

The effect is done without requiring extra HTML elements.

This collection allows changing the direction of the effect on different viewports. If this is not desired, it is recommended to use the [effects-hoverText] collection instead.

This collection is sepparate, as it requires a different syntax and CSS rules.

II. Initialization

Just download and refference the [ asterisk-effects-hvrText-responsive.css ] file. * Prefferably done in the <head> seaction

<link href="--myPath--/asterisk-effects-hvrText-responsive.css" type="text/css" rel="stylesheet">

III. Usage

Each Effect requires three declarations on the element :

1. hasHvrText-*pseudoElement* class - defining which pseudo-element will be used

2. hasHvrText-*direction* class - defines where the element will be positioned (can be changed for diff. viewports)

3. data-hvrText attribute - defining the text which will be shown on hover.

Format : <div class="hasHvrText-*pseudoElement* hasHvrText-*direction*" data-hvrText="Text-to-be-shown"> </div>

Example : <div class="hasHvrText-before hasHvrText-left" data-hvrText="Show me on hover!"> </div>

IV. Class-name Syntax

Each className has 2 parts :

1. static hasHvrText-* prefix

2. *-pseudoElement or *-direction suffix.

Item Values Description
*-pseudoElement before | after defines which pseudo-element will be used
*-direction left | right | top | btm defines where the element will be positioned

V. Responsive Utilities : Viewport-Prefixes

To create different scenarios for specific viewports, any hasHvrText-*direction* class can be prefixed with any *viewport*- or *viewport*-o- syntax.

Format : <div class=" hasHvrText-pseudoElement hasHvrText-direction *viewportPrefix*-hasHvrText-direction"> </div>
Example : <div class=" hasHvrText-before hasHvrText-top sm-hasHvrText-left"> </div>
// The effect will be : on 'top' for xs viewports - on 'left' for sm viewports or larger
Example : <div class=" hasHvrText-before hasHvrText-top sm-o-hasHvrText-left"> </div>
// The effect will be : on 'top' for xs viewports - on 'left' for sm viewports only - on 'top' for other viewports
Viewport values   sm-* md-* lg-* xl-*
Viewport-only values xs-o-* sm-o-* md-o-* lg-o-*  
<div class="hasHvrText-left-before"> </div>
Will take effect on all viewports (xs and higher)
<div class="sm-hasHvrText-left-before"> </div>
Will take effect on specified viewport (sm) and higher
<div class="sm-o-hasHvrText-left-before"> </div>
Will take effect on specified viewport (sm) only
