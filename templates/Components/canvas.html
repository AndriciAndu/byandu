The 'Canvas' Component

Components : Canvas

I. Description

Baseline canvas Component, used to create a animated background effect.

II. Initialization

Step 1. Download and refference the asterisk-component-canvas.js JS file.

<script src="--myPath--/asterisk-component-canvas.js" type="text/javascript"></script>

Step 2. Make the next function call :

asterisk.components.canvas.initAll();

III. Usage

Step 1. Add the Component's Markup to your Element :

* the markup should be placed as the first child, within your Element

Step 2. Change the bgCanvas-{type} class to apply the desired animation.

* Available classes : bgCanvas-bubbles | bgCanvas-confetti | bg-canvas-web

<div> <-- my Element --> <canvas class="bgCanvas bgCanvas-bubbles"> </canvas> <-- add other markup/children --> </div>

Step 3. To further customize, add any of the following HTML data-attributes :

data-canvas-particleMaxRadius = number - sets the particles' maximum size

data-canvas-particleColors = array of strings - sets the available colors

data-canvas-particleLimit = number - sets how many particles will be generated

<canvas class="bgCanvas bgCanvas-bubbles" data-canvas-particleMaxRadius="40" data-canvas-particleColors="[ '#2C3E50', '#E74C3C', '#ECF0F1' ]" data-canvas-particleLimit="300"> </canvas>

If any of these HTML data-attributes are ommited, the canvas will default to the values within asterisk.components.canvas.settings.default Object.

var canvasObject = {
... // these settings will affect all Canvases (if no more-specific values are assigned) default_settings : {
default_maxRadius : 40, default_particleLimit : 200, default_colorArray : [ '#2C3E50', '#E74C3C', ... ]
}, // these settings will affect all bgCanvas-bubbles Canvases (if no inline data-attribute values are assigned) bubbles : {
default_particleLimit : 300, default_colorArray : [ '#3498DB', '#2980B9', ... ]
}, // these settings will affect all bgCanvas-confetti Canvases (if no inline data-attribute values are assigned) confetti : {
default_particleLimit : 200, default_colorArray : [ '#55476A', '#AE3D63', ... ]
} ...
};
If the styling is set on multiple parameters, priority is set by the following specificity:
inline data-attributes >
canvasObject.*type*.*attribute* >
canvasObject.default_settings.*attribute* targetCanvas.maxRadius = targetCanvas.getAttribute('data-canvas-maxRadius') || canvasObject[current_type].default_maxRadius || canvasObject.default_settings.default_maxRadius;

*Note : Each canvas has it's size set through Javascript, to the full width/height of it's Parent Element.

As such, the Parent Element must have height (either through child elements other than the canvas, or set directly).

IV. Available Methods

canvas_pause() stops it's requestAnimationFrame callback

* it keeps the canvas' properties in memory.

* it will statically display the current painted screen.

canvas_resume() restarts the requestAnimationFrame callback.

canvas_disable() removes it's requestAnimationFrame callback

* it removes all the canvas' properties from memory.

* it will not display anything - clears the painted screen.

* in order for it to 'work' after calling canvas_disable(myCanvas), it must be initialized again with asterisk.components.canvas.init(myCanvas).

<canvas id="myCanvas" class="bgCanvas bgCanvas-bubbles"> </canvas>
var myCanvas = document.getElementById('myCanvas');
myCanvas.canvas_pause(); myCanvas.canvas_resume();
myCanvas.canvas_disable(); asterisk.components.canvas.init(myCanvas)
